# RPG 角色属性记录应用 — AI 开发指令

## 项目概述
这是一个类 RPG 角色属性记录应用，用户通过自由文本输入日志，AI 自动分析并转化为角色属性和技能变化。使用 React + TypeScript 开发。

## 开发规范
- 使用 TypeScript，确保类型安全
- 组件使用函数式组件 + Hooks
- 使用 Tailwind CSS 编写样式，暗色主题
- 状态管理使用 Zustand
- 所有数据存储在 IndexedDB（Dexie.js）
- AI 接口支持 Claude API / OpenAI API，用户自行配置 Key

## 代码风格
- 使用 ESLint 和 Prettier
- 组件名使用 PascalCase
- 函数名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE
- 类型定义统一放在 types/index.ts

## 测试要求
- 每个功能完成后手动测试
- 确保数据在 IndexedDB 中正确存储和读取
- 测试 AI 分析结果的 JSON 解析
- 测试用户修改 AI 建议值后数据正确更新
- 测试各种边界情况（空日志、超长文本、网络异常）

## 协作流程
- 逐步推进：每完成一个步骤后汇报，用户验证确认后再进行下一步
- 文件同步：每次更改完成后同步更新相关文件（代码、文档、类型定义等）
- 关键决策点：遇到架构选择、方案取舍、接口设计等关键点时，停下来与用户确认
- 问题处理：遇到问题时给出 2-3 个备选方案并说明利弊，由用户决定，不自行选择

## 注意事项
- 保持代码简洁，MVP 不做过度设计
- 优先实现核心闭环：输入 → AI 分析 → 确认 → 更新角色卡
- 确保移动端适配，触控友好
- AI 分析仅做建议，用户拥有最终解释权
- API Key 存储在本地，不上传任何服务器
